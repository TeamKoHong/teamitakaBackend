## 🔐 인증 테스트

### 1. 로그인
```bash
POST /auth/login
Content-Type: application/json

{
  "email": "test@example.com",
  "password": "password"
}
```

### 2. JWT 토큰 사용
로그인 후 받은 토큰을 Authorization 헤더에 포함:
```
Authorization: Bearer <your-jwt-token>
```

## 🧪 API 테스트 순서

### 1. 기본 조회 API (인증 불필요)
- ✅ `GET /user` - 사용자 목록
- ✅ `GET /projects` - 프로젝트 목록
- ✅ `GET /recruitments` - 모집공고 목록
- ✅ `GET /comment/00000000-0000-0000-0000-000000000002` - 댓글 조회

### 2. 인증 필요 API
1. **로그인하여 토큰 획득**
2. **토큰으로 다음 API 테스트:**
   - `GET /profile/00000000-0000-0000-0000-000000000001` - 프로필 조회
   - `GET /profile/00000000-0000-0000-0000-000000000001/ratings` - 평점 조회
   - `GET /profile/00000000-0000-0000-0000-000000000001/projects` - 사용자 프로젝트
   - `GET /reviews/user/00000000-0000-0000-0000-000000000001` - 사용자 리뷰
   - `GET /applications/00000000-0000-0000-0000-000000000002` - 지원자 목록

### 3. 프로젝트 관련 API
- `GET /projects/00000000-0000-0000-0000-000000000003/members` - 팀원 목록
- `GET /projects/00000000-0000-0000-0000-000000000003/timeline` - 타임라인
- `GET /projects/00000000-0000-0000-0000-000000000003/todo` - 할 일 목록

## 🚨 주의사항

1. **인증이 필요한 API**: JWT 토큰을 먼저 획득해야 합니다
2. **테스트 데이터**: 시드 데이터가 자동으로 생성됩니다
3. **ID 형식**: UUID 형식을 정확히 입력해야 합니다
4. **CORS**: 브라우저에서 테스트 시 CORS 정책을 확인하세요

---

## 📞 문제 해결

### 자주 발생하는 오류
- **401 Unauthorized**: JWT 토큰이 필요하거나 만료됨
- **404 Not Found**: 잘못된 ID 사용
- **500 Internal Server Error**: 서버 오류 (잠시 후 재시도)

### 도움말
- GitHub Actions CI/CD 로그에서 자동 테스트 결과 확인 가능
- 모든 API는 실제 프로덕션 환경에서 테스트됩니다
